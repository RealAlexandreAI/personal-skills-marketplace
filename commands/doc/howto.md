# Create How-to Guide Documentation

## Function
Create goal-oriented how-to guide documentation following the Diátaxis framework

## Trigger Condition
When user inputs `/howto`

## Behavior
Create a document that meets Diátaxis how-to guide standards, helping users complete specific tasks efficiently

## Diátaxis Classification
- **User State**: Working mode (applying existing knowledge)
- **Knowledge Type**: Action-oriented (task completion)
- **Purpose**: Problem solving and task completion

## How-to Guide Characteristics
- **User State**: Working mode, applying existing knowledge
- **Goal**: Task completion and problem solving
- **Approach**: Direct, actionable steps
- **Focus**: Results and efficiency
- **Tone**: Practical, direct, solution-focused

## Document Structure Template

```markdown
# How to [Complete Task]

## Goal
Step-by-step guide to complete [specific task]

## Prerequisites
Before starting, ensure you have:
- [Required tools]
- [Required data/permissions]
- [Basic environment requirements]

## Steps

### Step 1: [Step Name]
1. [Specific action 1]
2. [Specific action 2]
3. [Specific action 3]

**Expected Result**: [Expected outcome]

### Step 2: [Step Name]
[Repeat above structure]

## Verify Results
- [How to confirm task completion]
- [Expected results]
- [Success indicators]

## Troubleshooting
**Problem**: [Common issue 1]
**Solution**: [Specific solution method]

**Problem**: [Common issue 2]
**Solution**: [Specific solution method]

## Related Resources
- [Related documentation links]
- [Support resources]
- [Advanced operation guides]
```

## Writing Guidelines

### 1. Goal-Oriented
- Focus on task completion goals
- Emphasize results and efficiency
- Provide clear success criteria

### 2. Specific
- Provide detailed operational steps
- Include specific commands and parameters
- Give clear expected results

### 3. Practical
- Focus on solving real-world problems
- Provide immediately executable steps
- Include troubleshooting information

### 4. Efficient
- Help users complete tasks quickly
- Avoid unnecessary explanations
- Provide the most direct path

## Quality Checklist
- [ ] Clear task objectives
- [ ] Clear prerequisites
- [ ] Detailed operational steps
- [ ] Clear expected results
- [ ] Result verification methods
- [ ] Troubleshooting information
- [ ] Related resource links
- [ ] Practical tone

## Usage Example

```
/howto Create a guide for deploying applications to production environment
```

This will generate a guide to help users complete application deployment tasks, including detailed steps, verification methods, and troubleshooting information.
