# Create Tutorial Documentation

## Function
Create learning-oriented tutorial documentation following the Diátaxis framework

## Trigger Condition
When user inputs `/tutorial`

## Behavior
Create a document that meets Diátaxis tutorial standards, helping users learn new skills and knowledge through guided learning experiences

## Diátaxis Classification
- **User State**: Learning mode (acquiring new skills)
- **Knowledge Type**: Action-oriented (learning by doing)
- **Purpose**: Skill acquisition and education

## Tutorial Characteristics
- **User State**: Learning mode, acquiring new skills
- **Goal**: Education-oriented, skill development
- **Approach**: Step-by-step learning with explanations
- **Focus**: Learning process, not task completion
- **Tone**: Instructional, supportive, encouraging

## Document Structure Template

```markdown
# [Skill/Topic Name] Tutorial

## Learning Objectives
Through this tutorial, you will learn:
- [Specific skill 1]
- [Specific skill 2]
- [Specific skill 3]

## Prerequisites
Before starting, ensure you have:
- [Basic knowledge requirements]
- [Tool and environment requirements]
- [Permission requirements]

## Learning Steps

### Step 1: [Step Name]
**Learning Goal**: Understand/master [specific content]

**Actions**:
1. [Specific action 1]
2. [Specific action 2]
3. [Specific action 3]

**Key Learning Points**:
- [Important concept 1]
- [Important concept 2]

**Practice**: [Hands-on practice content]

### Step 2: [Step Name]
[Repeat above structure]

## Knowledge Summary
Through this tutorial, you have learned:
- [Skill summary 1]
- [Skill summary 2]

## Next Steps
- [Related tutorial 1]
- [Advanced topic 1]
- [Practice project suggestions]

## Common Questions
**Q: [Common question 1]**
A: [Detailed answer]

**Q: [Common question 2]**
A: [Detailed answer]
```

## Writing Guidelines

### 1. Learning-Oriented
- Focus on learning goals, not task completion
- Emphasize learning process, not final results
- Provide learning experiences, not just operational steps

### 2. Practical
- Include hands-on operations and exercises
- Provide opportunities for hands-on practice
- Learn through doing

### 3. Instructional
- Provide clear guidance
- Explain the purpose of each step
- Help understand principles and concepts

### 4. Progressive
- From simple to complex
- Gradually build knowledge and skills
- Ensure each step is understandable

## Quality Checklist
- [ ] Clear learning objectives
- [ ] Clear prerequisites
- [ ] Progressive learning steps
- [ ] Hands-on practice opportunities
- [ ] Knowledge summary
- [ ] Next steps guidance
- [ ] Common questions and answers
- [ ] Encouraging tone

## Usage Example

```
/tutorial Create a tutorial about React Hooks
```

This will generate a tutorial document to help users learn React Hooks, including learning objectives, prerequisites, step-by-step learning process, and practice opportunities.
