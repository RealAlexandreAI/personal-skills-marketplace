# Create Reference Documentation

## Function
Create information-oriented technical reference documentation following the Diátaxis framework

## Trigger Condition
When user inputs `/reference`

## Behavior
Create a document that meets Diátaxis reference standards, providing accurate and complete information for quick lookup

## Diátaxis Classification
- **User State**: Working mode (needing specific information)
- **Knowledge Type**: Understanding-oriented (information retrieval)
- **Purpose**: Information lookup and verification

## Reference Characteristics
- **User State**: Working mode, needing specific information
- **Goal**: Information retrieval and verification
- **Approach**: Comprehensive, structured data
- **Focus**: Accuracy and completeness
- **Tone**: Neutral, factual, authoritative

## Document Structure Template

```markdown
# [Topic] Reference

## Overview
Basic information and purpose of [topic]

## Syntax/Format
```
[Code example or format description]
```

## Parameters
| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| param1 | string | Yes | - | [Parameter description] |
| param2 | int | No | 0 | [Parameter description] |
| param3 | boolean | No | false | [Parameter description] |

## Return Values
[Return value type and description]

## Examples
```[language]
[Code example]
```

## Error Codes
| Error Code | Description | Solution |
|------------|-------------|----------|
| 1001 | [Error description] | [Solution] |
| 1002 | [Error description] | [Solution] |

## Related Topics
- [Related topic 1]
- [Related topic 2]
- [Related topic 3]

## Version Information
- Current version: [Version number]
- Last updated: [Date]
- Compatibility: [Compatibility information]
```

## Writing Guidelines

### 1. Accuracy
- Information is accurate and error-free
- Provide up-to-date data
- Verify all technical details

### 2. Completeness
- Complete content with no omissions
- Include all parameters and options
- Provide complete examples

### 3. Logical Structure
- Organize by logical structure
- Use consistent formatting
- Easy to scan and find information

### 4. Scannability
- Easy to scan and find information
- Use clear headings
- Provide indexes and cross-references

## Quality Checklist
- [ ] Accurate technical information
- [ ] Complete parameter descriptions
- [ ] Clear structural organization
- [ ] Scannable format
- [ ] Working examples
- [ ] Error handling information
- [ ] Version and compatibility information
- [ ] Related topic links

## Usage Example

```
/reference Create a technical reference document for API endpoints
```

This will generate a reference document containing complete information about API endpoints, including parameters, return values, examples, and error handling.
